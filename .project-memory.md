# Maila Mod 项目记忆

## 项目信息

- **Mod ID**: `FuYn.Maila`
- **目标**: 修改卡牌提示项（card keywords）等提示项的文本显示。
- **入口**: `MainFile.cs` — `[ModInitializer]` 标记，`Initialize()` 中通过 `harmony.PatchAll()` 自动应用所有 Patch。

## 已实现的 Patch

### `HoverTipPatcher` (`Patches/HoverTipPatcher.cs`)

- **对应需求**: 在描述中附加命名空间和类名。
- **Patch 类型**: Postfix
- **目标方法**:
  - `HoverTipFactory.FromKeyword(CardKeyword)`: 针对枚举 Keyword，附加 `MegaCrit.Sts2.Core.Entities.Cards.CardKeyword.{Value}`。
  - `PowerModel.DumbHoverTip` (Getter): 针对 Power，附加类全名。
  - `RelicModel.HoverTips` (Getter): 针对 Relic，附加类全名（到第一个提示项）。
  - `PotionModel.HoverTip` (Getter): 针对 Potion，附加类全名。
  - `AfflictionModel.HoverTips` (Getter): 针对 Affliction，附加类全名。
  - `EnchantmentModel.HoverTips` (Getter): 针对 Enchantment，附加类全名。
- **实现细节**: 使用反射修改 `HoverTip` 结构体的私有字段 `<Description>k__BackingField`。由于 `HoverTip` 是结构体，需要处理装箱/拆箱逻辑。

## 关键链路（card_keywords 显示流程）

```
card_keywords.json
  → LocManager.LoadTable (解析 JSON)
  → LocTable (存储)
  → CardKeywordExtensions.GetTitle/GetDescription (创建 LocString)
  → HoverTipFactory.FromKeyword (创建 HoverTip，缓存)
      → HoverTipPatcher (Postfix) 修改 Description，附加类型全名
  → HoverTip 构造时调用 LocString.GetFormattedText()
      → LocString.GetRawText()
      → SmartFormat 格式化
```
